📊 Umami – NAV ResearchOps

Åpen kildekode produktanalyseverktøy for sporing og analyse av brukerdata på NAVs nettsider og apper.

Team: @navikt/team-researchops · Versjon: Umami v3.0.3 · Runtime: Next.js (Node 22)

1. Hva er denne applikasjonen?

umami er NAVs deployment av Umami – en selvhostet, personvernvennlig webanalyseplattform. Applikasjonen samler inn anonymiserte sidevisnings- og hendelsesdata fra NAVs tjenester og presenterer dem i et brukervennlig dashboard.

I tillegg inneholder repoet et sett med tilpassede sporingsskript som erstatter standard Umami tracker. Disse skriptene har innebygd redaksjon (masking) av sensitiv informasjon som NAV-identer og UUIDer.

Nøkkelegenskaper:
  • Ingen informasjonskapsler – fullstendig GDPR-vennlig
  • Kun landsnivå geo-lokalisering (bruker dbip-country-lite.mmdb, ingen bydata)
  • Innebygd redaksjon av NAV-identer, veilederidenter og UUIDer i URL/tittel/referrer
  • PostgreSQL- eller ClickHouse-databasestøtte
  • Kjører som Docker-container på NAIS

3. API-endepunkter

3.1 Datainnsamling (Tracking)

MetodeEndepunktBeskrivelse
POST /api/send Hovedendepunkt for å sende sidevisnings- og hendelsesdata. Brukes av sporing.js og de fleste sporingsskript-varianter.
POST /umami Alternativt innsamlingsendepunkt brukt av umamisporing.js (legacy/alternativ rute).

3.2 Umami REST API (innebygd)

Umami v3 eksponerer et komplett REST API for administrasjon og datauttak. De viktigste gruppene:

MetodeEndepunktBeskrivelse
POST/api/auth/loginAutentisering – returnerer JWT-token
GET/api/websitesHent liste over registrerte nettsteder
GET/api/websites/:id/statsHent statistikk (sidevisninger, besøkende, bounce rate, etc.)
GET/api/websites/:id/pageviewsSidevisninger over tid
GET/api/websites/:id/eventsEgendefinerte hendelser over tid
GET/api/websites/:id/metricsMetrikker (URL-er, referrere, nettlesere, OS, land, etc.)
GET/api/websites/:id/activeAntall aktive besøkende akkurat nå
POST/api/websitesOpprett et nytt nettsted
GET/api/usersAdministrer brukere (kun admin)
GET/api/teamsAdministrer team

4. Datamodell

4.1 Tracking payload (POST /api/send)

Request body som sendes fra sporingsskriptet til serveren:

{
  "type": "event" | "identify",
  "payload": {
    "website": "uuid-for-nettstedet",
    "hostname": "www.nav.no",
    "screen": "1920x1080",
    "language": "nb-NO",
    "title": "Sidetittel",
    "url": "/aktuell/side",
    "referrer": "https://www.google.com/",
    "tag": "optional-tag",
    "name": "event-navn (kun ved hendelser)",
    "data": { "key": "value" },
    "id": "valgfri bruker-id"
  }
}
FeltTypeBeskrivelse
typestring"event" for sidevisning/hendelse, "identify" for brukeridentifisering
payload.websitestring (UUID)Nettstedets unike ID i Umami
payload.hostnamestringVertsnavnet til siden
payload.screenstringSkjermoppløsning, f.eks. 1920x1080
payload.languagestringNettleserens språk
payload.titlestringSidetittel (document.title)
payload.urlstringNåværende URL-sti
payload.referrerstringReferrer-URL
payload.tagstring?Valgfri tag satt via data-tag
payload.namestring?Hendelsesnavn (for egendefinerte events)
payload.dataobject?Vilkårlig metadata knyttet til hendelsen
payload.idstring?Bruker-ID for identifisering

4.2 Response

{
  "cache": "kryptert-cache-token",
  "disabled": false
}
FeltTypeBeskrivelse
cachestringCache-token som sendes tilbake som x-umami-cache header ved neste request
disabledbooleanHvis true, stopper klienten all videre sporing

4.3 Databasemodell (Prisma / PostgreSQL)

Umami bruker Prisma ORM. De viktigste tabellene:

TabellBeskrivelseNøkkelfelt
websiteRegistrerte nettstederid, name, domain, created_at
sessionBesøksøkter (fingerprint-basert)id, website_id, hostname, browser, os, device, screen, language, country
website_eventAlle sidevisninger og hendelserid, website_id, session_id, url_path, referrer_path, event_name, event_type (1=sidevisning, 2=hendelse), created_at
event_dataEgendefinert metadata for hendelserid, website_event_id, event_key, string_value, number_value, date_value, data_type
userUmami-brukere (for dashboard-innlogging)id, username, role, created_at
teamTeam som kan dele nettstederid, name, access_code
team_userKobling mellom team og brukereteam_id, user_id, role
team_websiteKobling mellom team og nettstederteam_id, website_id

5. Sporingsskript-varianter

Applikasjonen serverer flere varianter av sporingsskriptet med ulik grad av personvernbeskyttelse:

SkriptEndepunktRedaksjon
sporing.js /api/send Ingen – standard Umami v3 tracker
umamisporing.js /umami Ingen – alternativ rute
sporing-uten-navident.js /api/send NAV-ident → [REDACTED-NAVIDENT]
sporing-uten-veilederidentid.js /api/send Veilederident → [REDACTED-VEILEDERIDENT]
sporing-uten-uuid.js /api/send UUID → [REDACTED-UUID]
sporing-uten-uuid-og-navident.js /api/send UUID → [REDACTED-UUID] NAV-ident → [REDACTED-NAVIDENT]
sporing-navet6.js /api/send Lesbar/ubehandlet variant for Navet 6

5.1 Integrasjon – eksempel

<script
  defer
  src="https://<umami-host>/sporing/sporing-uten-uuid-og-navident.js"
  data-website-id="din-website-uuid"
></script>

5.2 Script-attributter

AttributtPåkrevdBeskrivelse
data-website-id✅UUID for nettstedet i Umami
data-host-url❌Override for Umami-serverens URL
data-tag❌Tag som legges til alle hendelser
data-auto-track❌Sett til "false" for å slå av automatisk sporing
data-do-not-track❌Sett til "true" for å respektere DNT-header
data-domains❌Kommaseparert liste med domener som tillater sporing
data-exclude-search❌Sett til "true" for å fjerne query-parametere
data-exclude-hash❌Sett til "true" for å fjerne hash fra URL
data-before-send❌Global funksjon som kjøres før sending

5.3 JavaScript API

// Spor en egendefinert hendelse
umami.track("knapp-klikket", { kategori: "skjema", verdi: 42 });

// Spor en sidevisning manuelt
umami.track();

// Identifiser bruker (med data)
umami.identify({ rolle: "saksbehandler" });
