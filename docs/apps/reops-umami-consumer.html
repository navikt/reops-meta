🔄 reops-umami-consumer

Kafka-consumer som leser analytics-events, kjører personvernfiltrering, og videresender dem til Umami.

Arkitektur

Kafka-topic
→
KafkaService
→
FilterService
→
UmamiService
→
Umami API
/api/send
  • Kafka-topic – Mottar rå analytics-events (JSON).
  • KafkaService – Lytter på topic, deserialiserer til Event, og orkestrerer flyten.
  • FilterService – Personvernfiltrering: strukturelle regler, URL-policy og regex-basert redaksjon.
  • UmamiService – Sender det filtrerte eventet videre til Umami via HTTP POST.

Personvernfilter (FilterService)

Filteret kjører i tre uavhengige lag:

1. Strukturelle regler (nøkkelbasert)

NøkkelHandlingResultat
ip_addressDROPFeltet fjernes
ipErstattes"$remote"
api_key, device_id, websiteBevaresUendret
Advertising-ID-er (idfa, gaid, …)Anonymisert"[PROXY]"

2. URL-policy (kontekstbasert)

payload.url og payload.referrer behandles som URL-er. Inne i payload.data gjenkjennes nøkler som url, href, path, pathname, destination, lenkesti, m.fl. og håndteres med URL-sanitering.
