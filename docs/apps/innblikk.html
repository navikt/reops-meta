🚀 Innblikk — Innblikk Frontend

Webanalyse og produktinnsikt for NAV — bygget på Umami + BigQuery

Hva er Innblikk?

Innblikk er NAVs interne verktøy for webanalyse. Applikasjonen gir produktteam innsikt i brukeradferd gjennom ferdigbygde dashboards, hendelsesutforsking, trafikkanalyser, traktanalyser, brukerreiser og personvernssjekker — alt uten å gå på bekostning av brukervennlighet, datasikkerhet eller personvern.

Data lagres i BigQuery (GCP) og Umami-instansen, og tilgjengeliggjøres via en Express-server som proxy-er mot backend, Siteimprove og BigQuery.

Sikkerhet og autentisering

  • Azure AD / @navikt/oasis: Alle /api/bigquery og /api/backend ruter krever autentisering via Azure AD-token.
  • OBO-token: Backend-proxyen utveksler brukerens token med On-Behalf-Of (OBO) mot start-umami-backend.
  • SQL-validering: SQL-editoren tillater kun SELECT/WITH-spørringer. DML/DDL (INSERT, UPDATE, DELETE, DROP, etc.) blokkeres.
  • BigQuery-kostnadsgrense: Alle spørringer har en maks grense på 500 GB skannet data.
  • Helmet: HTTP-sikkerhetshoder satt via Helmet-mellomvare.
  • Revisjonslogg: Alle BigQuery-spørringer logges med NAV-ident og analysetype.
